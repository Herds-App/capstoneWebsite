Installation Instructions

  1. Ensure that you have an updated Java Version on your machine "java --version" (23.0.1)
  2. If you see the option between Windows, MacOS, and Linux Select your platform, otherwise move on to step 3
  3. Ensure you have Visual Studio Code installed.
  4. Click on Android as the type of application.
  5. Follow the instructions to Install the Flutter SDK.
    1. Download the flutter installation bundle
    2. Create a folder at C:\Users\{username} to install flutter to
    3. Extract the flutter installation bundle to this location
    4. Add the location\dev\flutter\bin to your PATH variables
  6. Follow the instructions to Configure Android development.
    1. Download Android Studio Ladybug.
    2. Launch Android Studio and follow the setup wizard
    3. Install the following components:
      • Android SDK Platform, API 35.0.1
      • Android SDK Command-line Tools
      • Android SDK Build-Tools
      • Android SDK Platform-Tools
      • Android Emulator
  7. Follow the instructions to Configure your target Android device.
    1. Enable VM acceleration on your device
    2. Go to the More Actions dialog and click SDK Manager
    3. Ensure “Show Package Details” is clicked
    4. Click on the check mark next to Android 15 (“VanillaIceCream”)
    5. Click apply and wait for the download to finish
    6. Close out of the settings dialog
    7. Got to the More Actions dialog and click Virtual Device Manager
    8. Click the plus button
    9. Select Medium Phone and click next
    10. Select VanillaIceCream as the system image and click next
    11. Click finish
    12. In the VS code terminal run “flutter doctor –android-licenses” and agree to all the licenses
  8. Follow the instructions to Check your development setup.
    1. Run “flutter doctor” and ensure that the following have check boxes
      • Flutter
      • Windows Version
      • Android Studio
      • Connected Device
      • Network resources
    2. If there are any issues troubleshoot using the flutter documentation
  9. Pull the latest release from HerdsApp Repo.
    1. Access the GitHub and download the latest release.
    2. If you do not have access to this repo, send robonham@ttu.edu an email with your github username, and he will give you access.
  10. Enter the repo folder using VS code.
  11. In the bottom right corner of VS code, once you open a .dart file you will see “No device” Click on this and select (cold boot) Medium Phone.
  12. Type “flutter run” into your terminal.

User Manual

When opening Herds, the User will be prompted for their login information and if they do not have an account, there will be an option to register for one below. When registering for an account, the user will be prompted for their email address, phone number and password. This will ensure that the User’s information will remain secure and only accessible to them. Refresh and access tokens are also utilized so that once a User has logged in, they have the option to remain logged in, so they do not have to repeat the process above. The User’s profile will be public so anyone can see it, and the profile can contain whatever the User would like. For example, it will contain a profile picture, display name and username, organizations the User is a member of (Herds), as well as posts that the User has created.

Once logged in, you will be moved to the home page where the User will be able to see different feeds, and these feeds are attached to different Herds that they are a member of. The User will be able to see their own posts within the feeds, as well as other Users that are a member of the same Herd. A User will be able to tell which Herd they are currently in by the white highlighted tab at the top of the page. To create a new post within a Herd, the User will simply press the plus button towards the top of the screen, giving a prompt to fill with their message. Before this post is sent, it will be moderated to ensure the content of the post is appropriate. The User also has the option to “like” a post if they enjoyed the content of the post and leave a comment on said post.

If the User decides to click on the Herds tab, they will be able to see all the different Herds that they are a member of. When the User clicks on these Herds, it will open up a calendar, allowing them to see different events scheduled throughout the semester. This will range from University Herds, where it will show events the University is hosting, as well as Herds created by Students, where you will be able to see scheduled study hall hours, events, dress codes, RSVP for events and if necessary pay for events through the app. If the User happens to be the admin for a Herd, they will be able to add events to the calendar. Some Herds will be public and private, if public you would not need to be a member of the Herd to see posts and the calendar, however you would need to subscribe to the Herd and you would also not be able to create posts within that Herd or communicate via direct or group messaging within that Herd (Would also have the option to create a chat with members of the Herd, but would not be able to communicate directly in the Herd). If it is private, then you would need to be a member of the Herd to have access to what is listed above. In the Herds tab, if the User presses on the plus button they will be able to create a new Herd, and the name of it is up to the User’s preference. Once this new Herd is created, the User can then go back to the Home page where they will be able to see this new Herd highlighted in white at the top. Once there are posts from other members (or themselves) the User will be able to see this in the feed of that Herd.

In the Explore page, the User will be able to search for new Herds that they would like to be a member of. This is assuming that the Herd is public, if it is private the User will have to be invited to join the Herd, or request for permission to join the Herd. Through the Explore page, the User will also be able to search for things like University, GameDay and External Herd feeds. These things are fairly self-explanatory, they are information related to a given University, GameDay scores and when the games are scheduled, etc. and information that has been posted outside of a Herd. The User will also be able to search the Emergency System, to find any emergencies in their area that they should avoid.

Through our Event Handling System, we have the Calendar (explained above), Notification System and Study Hall feature. The User has the option to enable the Notification System, giving them updates on their Herds whether it is about new posts or changes to the Herd calendar. The Study Hall feature will enable students to schedule study hall hours within the Herd for their classes, allowing others to see it on the Herd calendar if other Users would like to join. Mentioned above there is also Payment, the User will be able to pay for events through Herds that are scheduled to the Herds calendar (that they have RSVPd for) through a secured payment system.

Within a Herd, a User will be able to create a Direct or Group message with other members of their Herd. This can be done simply by pressing on a user’s display name and selecting the option to create a direct message. If the User would like to add other members to this chat, then they would also click on their display name and add them to the already created chat. If sending a message to a User that is not a member of your Herd then you would repeat the steps listed above, however the User that is receiving the message would need to accept this message. This is to ensure that the User is comfortable receiving a message from a User that they do not know.

For certain features such as messaging, aspects of the Explore page and Event Handling System will utilize activity tracking. This will allow the User to see more content that they would like, ensuring the User has a pleasant experience when using Herds. Mentioned above, some aspects of Herds will be moderated to ensure that no inappropriate content slips through to dampen the Users experience. Herds will also be able to support iOS and Android devices, allowing people who own either device to utilize the app.
